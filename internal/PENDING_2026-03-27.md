## P1 — High priority (before supervisor review v1.0)

### Metrology standards (writing tasks, no experimental redesign)

- [ ] **Add GUM JCGM 100:2008 to METHODOLOGY §2.4.**  
  Classify every uncertainty source as Type A (statistical) or Type B (datasheet/calibration). Report combined result as expanded uncertainty U = k·u_c with k=2 (~95% coverage). Currently the table lists magnitudes without combining them or declaring k.

- [ ] **Declare metrological traceability status of YuMi.**  
  YuMi AT = 1.36 mm is Type B from ABB ISO 9283 spec — not a traceable calibration result. If IQS unit has no current calibration certificate with SI chain, declare this explicitly in METHODOLOGY §2.4 and README §9 Known Limitations. One sentence is enough.

- [ ] **Add IEEE Std 952-2020 to METHODOLOGY §1.1 and RESEARCH_PLAN §9 refs.**  
  Standard for gyroscope characterization via Allan Variance. Complements IEEE Std 1293 already cited. Required for IEEE TIM submission.

- [ ] **Specify chi-squared CI for AVAR coefficients in METHODOLOGY §1.3.**  
  AVAR CI comes from the chi-squared distribution of the overlapping estimator, not Gaussian bootstrap. Specify this explicitly. Report k=2.

- [ ] **Add ISO 5725 framing to METHODOLOGY §3.1 and §3.5.**  
  Declare scope: study measures within-session repeatability and cross-session intermediate precision (ISO 5725-3 term). Not full reproducibility. Two sentences.

- [ ] **Add AVAR lower-bound argument to RESEARCH_PLAN §3.6.**  
  Static AVAR is a lower bound on operational noise (Lethander & Taylor, 2023 — REFERENCES_POOL ref 2). This is the physical justification for why M2 < M3 < M4 progressively closes the gap. Currently implicit; make it explicit.

- [ ] **Declare scope of validity for M4 in METHODOLOGY §5.6.**  
  Velocity range (T1–T2 fitting, T3 held-out), temperature range (warm-up steady-state), angular rate range. Extrapolation beyond these ranges is not validated.

### Statistical / methodological

- [ ] **Pre-specify outlier rejection protocol.**  
  What happens when a SLAM backend diverges during a repetition? Include or exclude? With what criterion? Must be written before data collection. Add to METHODOLOGY §3.1.

- [ ] **Add stationarity verification for IMU static logs.**  
  AVAR assumes stationarity. If thermal drift during the session breaks this, coefficients are artifacts. Specify ADF or KPSS test as pre-condition before AVAR fitting. Add to METHODOLOGY §1.2.

- [ ] **Formal measurand definition.**  
  Add one paragraph to RESEARCH_PLAN §1 or METHODOLOGY §3 defining the measurand explicitly: "the mean difference in ATE RMSE between metrological simulation (M) and real hardware (R) under trajectory T2, per backend." Required vocabulary for a metrology journal.

- [ ] **Add waypoint precision as exploratory endpoint in T3.**  
  ATE is an indirect proxy for navigation performance. Waypoint position error (already feasible with T3 protocol and YuMi waypoints) is a task-relevant secondary metric. Add as exploratory endpoint in METHODOLOGY §3.5.

- [ ] **Strengthen physical interpretation of M4 variance terms.**  
  Each coefficient (c_ω, c_a, c_j, etc.) needs a physical mechanism sentence. Currently stated as "captures g-sensitivity" without quantitative grounding. Relevant for IEEE TIM reviewers.

- [ ] **Define warm-up policy for WT901C and D455 IMU.**  
  Mid-360 has a documented 20-min protocol. WT901C and BMI055 warm-up times are not specified. Add to EXPERIMENTAL_DESIGN §2.1.

- [ ] **Increase hand-eye calibration to ≥30 poses (orientation-diverse).**  
  Current protocol specifies ≥15 poses. 30+ with full orientation diversity reduces systematic hand-eye residual. Update EXPERIMENTAL_DESIGN §10.

## P2 — Medium priority (before manuscript draft)

- [ ] **Add verification vs validation distinction to RESEARCH_PLAN §1.**  
  The study performs *validation* (does the model represent reality?) not *verification* (is the model implemented correctly?). Use this vocabulary — it is standard in simulator V&V literature (AIAA G-010, DoD VV&A framework) and makes the contribution clearer to reviewers outside robotics.

- [ ] **Specify axis definition for CW/CCW asymmetry analysis (H2).**  
  Which axis defines CW vs CCW? Must be fixed before sessions run. Add to EXPERIMENTAL_DESIGN §7.
